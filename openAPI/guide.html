개발 가이드

기본 안내

YES24 Open API는 HTTP GET 방식을 사용하는 RESTful API입니다. 모든 요청의 Base URL은 다음과 같습니다.

https://openapi.yes24.com/api/v1

인증 (Authentication)

API를 호출하기 위해서는 apiKey가 필수입니다. 쿼리스트링(QueryString)에 포함하여 요청합니다.

curl -X GET "https://openapi.yes24.com/api/v1/search?apiKey=YOUR_API_KEY&keyword=달러구트" \
  -H "Accept: application/json"
기본 발급 키는 1일 5,000회 호출로 제한됩니다. 초과 시 상업적 제휴 문의를 부탁드립니다.

요청 및 응답 포맷

응답 포맷은 output 파라미터를 통해 json 또는 xml로 지정할 수 있으며, 기본값은 JSON입니다.

{
  "version": "20250101",
  "totalResults": 125,
  "startIndex": 1,
  "itemsPerPage": 10,
  "item": [
    { ...도서 정보... }
  ]
}

에러 코드 가이드

HTTP Status Error Code 설명
200 - 정상 처리
400 MissingParameter 필수 파라미터(예: keyword) 누락
401 InvalidApiKey 유효하지 않거나 만료된 API Key
429 RateLimitExceeded 일일 호출 허용량(5,000회) 초과
500 InternalServerError 서버 내부 오류 (YES24 측 문제)

API Reference

v1.0.0 (OAS3)
GET /api/v1/search 상품 검색 API

키워드를 기반으로 도서를 검색합니다.

Parameters

Name Description
apiKey
* required		 발급받은 인증키 (String)
keyword
* required		 검색어 (UTF-8 인코딩)
sort 정렬 방식 (accuracy: 정확도순(기본), sales: 판매량순, recent: 신간순)
maxResults 한 페이지에 출력될 결과 수 (기본: 10, 최대: 50)

Responses

{
  "version": "20250101",
  "totalResults": 1,
  "item": [
    {
      "itemId": 90365124,
      "title": "달러구트 꿈 백화점",

      "pubDate": "2020-07-08",
      "priceStandard": 13800,
      "cover": "https://image.yes24.com/goods/90365124/L",

GET /api/v1/list/bestseller 베스트셀러 목록 API

현재 실시간 베스트셀러 목록을 반환합니다.

상세 내용은 검색 API와 동일한 포맷을 제공합니다.
GET /api/v1/product/{itemId} 상품 상세 API

상품 고유번호(itemId)로 상세 책소개 및 목차를 조회합니다.

상세 내용은 검색 API 응답 + description, toc 필드가 추가됩니다.

자주 묻는 질문 (FAQ)

API 이용에 관해 가장 많이 들어오는 질문들을 모았습니다.

기본적으로 발급되는 API Key는 1일 최대 5,000회까지 호출이 가능합니다. 한도는 매일 자정(00:00 KST)에 초기화됩니다. 호출 한도를 증설해야 하는 기업 고객은 별도의 B2B 제휴 신청을 통해 한도를 상향할 수 있습니다.
네, 개인 및 비상업적 용도는 약관 동의 후 즉시 무료로 사용 가능합니다.
다만, YES24의 데이터를 기반으로 한 직접적인 영리 창출 서비스(수익형 앱, 쇼핑몰 연동 등)의 경우 API 사용 목적 검토 후 사용이 승인됩니다. 자세한 내용은 고객센터로 문의 바랍니다.
네, 응답 JSON/XML 내에 cover 필드를 통해 도서 표지 이미지 URL (https://image.yes24.com/...)을 제공합니다. 해당 URL을 서비스 내 `` 태그에 직접 연결하여 사용하실 수 있습니다.
출판사에서 제공하지 않은 데이터(예: 목차, 역자 정보 등)나, e-book 전용 필드 등 상품의 성격에 따라 일부 메타데이터 필드는 값이 없거나 null로 반환될 수 있습니다. 프론트엔드 연동 시 방어 코드(Optional Chaining 등) 작성을 권장합니다.
