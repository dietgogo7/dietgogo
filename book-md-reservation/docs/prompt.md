# AI 프롬프트 기반 개발 프로세스 (바이브코딩)

## 요약
본 문서는 "바이브코딩(Vibe Coding)" 방식을 활용하여 예스24 MD 미팅 예약 시스템을 개발한 전체 프로세스를 **AI 프롬프트 기반 개발 프로세스** 형태로 정리한 기술 문서입니다. 기획 단계부터 UI/UX 설계, 데이터 모델링, 세부 기능 구현에 이르기까지 AI와 상호작용하기 위해 사용된 핵심 프롬프트와 프롬프트 엔지니어링 전략을 요약했습니다.

## 문제분석
기존의 전통적인 개발 방식은 [요구사항 분석 ➔ UI/UX 설계 ➔ 마크업 ➔ 프론트엔드/백엔드 로직 구현]의 직렬적인 단계를 거쳐 시간과 리소스 소모가 컸습니다. 
본 프로젝트에서는 복잡한 인터랙션이 요구되는 **단계별 예약 폼(Step-by-step Form)**, **동적 스케줄링 및 공휴일 예외 처리**, **관리자(Admin) 페이지**, **다크모드 지원** 등의 요구사항을 빠르고 유연하게 구현하기 위해 지속적으로 AI에게 명령을 내리고 코드를 다듬어가는 '바이브코딩' 방식을 도입했습니다. 이를 성공적으로 이끌기 위해서는 추상적인 지시가 아닌 개발 단계별로 명확하고 구조화된 프롬프트가 필요했습니다.

## 해결전략
AI가 프로젝트의 컨텍스트를 잃지 않고 정확한 코드를 출력하도록, 개발 단계를 5단계로 세분화하고 각 단계별 목적에 맞는 특화 프롬프트를 작성하여 적용했습니다.

### 1. 개발 목표
예스24 출판사 영업 담당자와 예스24 MD 간의 미팅 스케줄을 원활하게 예약 및 관리할 수 있는 반응형 웹 기반의 **사용자용 미팅 예약 시스템**과 **관리자 페이지**를 단기간 내에 구축하는 것을 목표로 설정했습니다. 제약 조건으로 최신 프레임워크(React 등)를 쓰지 않고 **jQuery, ES5, 순수 CSS** 기반의 보수적인 환경을 유지하면서도 모던한 UX(Single Page Application 느낌의 화면 전환, 다크모드 등)를 제공하는 것을 목표로 헸습니다.

### 2. 요구사항 정의 단계
시스템의 전체적인 뼈대를 잡기 위해 AI에게 역할(Role)을 부여하고, 필수 기능과 유저 플로우(User Flow)를 텍스트로 정의하여 기능 명세서를 도출했습니다.

*   **사용된 프롬프트 예시:**
    > "너는 15년 차 시니어 기획자이자 웹 개발자야. 예스24 MD 미팅 예약 시스템을 만들려고 해. 프론트엔드 환경은 React나 Vue 없이 순수 HTML, CSS, jQuery(ES5)만 사용해야 해. 
    > 사용자는 **[MD 선택 ➔ 날짜 선택 ➔ 시간 선택 ➔ 예약자 주체 정보 입력 ➔ 완료]** 순서로 진행해야 해. 관리자는 예약 내역을 보고 승인/반려할 수 있어. 이 요구사항을 바탕으로 누락된 에지 케이스(Edge Case)를 분석하고, 화면 단위의 기능 명세서를 마크다운으로 정리해 줘."

### 3. UI 설계 단계
시각적 요소와 퍼블리싱 구조를 잡기 위해 브랜드 가이드라인을 제시하고, 초기 레이아웃 구조를 요청했습니다. 일부 단계에서는 AI 디자인 생성 도구(Stitch MCP)를 활용하여 시각적 청사진을 미리 확보했습니다.

*   **사용된 프롬프트 예시 (사용자 환경 & 다크모드):**
    > "예스24의 브랜드 컬러(블루 계열)를 기반으로 사용자 예약 페이지와 관리자(Admin) 대시보드의 UI 레이아웃을 작성해 줘. 모바일 환경을 고려한 반응형(Responsive) 디자인이어야 해. 추가로 웹사이트 상단에 토글 버튼을 두고, 사용자 기기 설정 및 버튼 클릭에 따라 동적으로 스타일이 바뀌는 **'다크모드'**를 적용하기 위해 CSS 변수(:root) 세팅 코드를 작성해 줘."
    
*   **사용된 프롬프트 예시 (어드민 레이아웃 조정):**
    > "어드민 페이지에서 좌측 LNB(메뉴)를 제외한 우측 콘텐츠 영역이 브라우저 해상도 변경에 상관없이 남은 전체 너비(Full-width)에 꽉 차도록 기존 HTML 구조와 CSS를 수정해 줘."

### 4. 데이터 구조 설계
프론트엔드 로직 구현 전, 데이터의 흐름을 통제하기 위해 JSON 기반의 Mock 데이터 모델링과 공휴일, 영업시간 등 핵심 설정 파일(Config)의 아키텍처를 AI에게 설계하도록 지시했습니다.

*   **사용된 프롬프트 예시 (데이터 및 스케줄링 로직):**
    > "백엔드 API가 아직 없으므로, 프론트엔드에서 사용할 Mock JSON 데이터를 설계해 줘. MD 목록, 예약 내역 데이터 구조명세서가 필요해. 
    > 또한, 예약 가능 시간 검증 로직을 위해 요일별 업무 시간 지정과 대한민국 법정 공휴일 예외 처리가 필요해. 하드코딩을 피하기 위해 동적 스케줄링 규칙을 담은 `schedule_config` 객체 구조와 휴일 데이터를 분리한 `holidays.js` 모듈 구조를 제안해 줘."

### 5. 프론트 구현
기본 마크업과 데이터가 준비된 후, 자바스크립트를 이용해 동적인 인터랙션과 고도화된 UX 처리를 단계별 리팩토링(Refining) 프롬프트를 통해 구현해 나갔습니다.

*   **사용된 프롬프트 예시 (플로우 개선 및 애니메이션 연동):**
    > "기존 코드는 예약 스텝이 아래로 계속 길어지는(Append) 방식인데 UX가 좋지 않아. 이를 화면 전환 애니메이션을 동반한 **단계별 마법사(Step-by-step) 흐름**으로 리팩토링해 줘.
    > 1. 각 단계별(MD 선택, 날짜 선택 등)로 섹션을 보이거나 숨길 때 CSS 트랜지션을 적용할 것.
    > 2. 각 스텝 하단에 '이전 화면으로 가기', '취소' 네비게이션 버튼을 추가할 것.
    > 3. 상단에는 현재 어느 스텝에 있는지 보여주는 시각적 진행 상태(Progress Indicator) 닷(점) UI를 추가하고 JS 로직으로 상태를 제어하게 해 줘."

## 코드
위 '4단계 데이터 구조 및 5단계 프론트 구현'에서 바이브코딩 대화를 통해 도출된 핵심 설정 모듈(공휴일 및 스케줄링) 코드의 결과물 예시입니다.

```javascript
// holidays.js (AI를 통해 분리 및 모듈화된 공휴일 설정 파일)
var HOLIDAYS = [
    "2026-03-01", // 3절
    "2026-05-05", // 어린이날
    // ... 휴일 데이터
];

// admin.js (AI가 개선한 스케줄 환경 설정 객체)
var SCHEDULE_CONFIG = {
    workDays: [1, 2, 3, 4, 5], // 월~금
    businessHours: { start: "09:00", end: "18:00" },
    lunchTime: { start: "12:00", end: "13:00" },
    meetingInterval: 30, // 30분 단위
    
    // 주어진 날짜가 예약 가능한 날인지 판별하는 함수 (프롬프트로 생성된 로직)
    isAvailableDate: function(dateString) {
        var dateObj = new Date(dateString);
        var day = dateObj.getDay();
        
        // 1. 주말 체크
        if (this.workDays.indexOf(day) === -1) return false;
        
        // 2. 공휴일 체크
        if (HOLIDAYS.indexOf(dateString) !== -1) return false;
        
        return true;
    }
};
```

## 적용방법
1. **신규 기능 추가 시**: 위 프로세스와 동일하게 요구사항 ➔ UI 설계 ➔ 데이터 구조 ➔ 기능 구현 순서로 AI 파트너에게 맥락을 제공하는 순차적 프롬프팅을 활용합니다.
2. **리팩토링 시**: 기존 코드를 복사하여 AI에게 제공한 뒤, "기존 로직과 호환성 유지", "ES5 문법 준수", "클래스명 보존" 등의 엄격한 제약 조건을 명시(`Prompt Engineering System Message`)하여 안정성을 확보하면서 점진적 적용을 진행합니다.
