# 도서 MD 미팅 예약 시스템 - 서비스 기획 명세서

## 1. 서비스 개요
- **서비스 목적**: 예스24 출판사 및 영업 담당자와 예스24 MD(상품기획자) 간의 미팅을 원활하게 조율하고 효율적으로 일정을 관리하기 위해 구축된 예약 시스템입니다.
- **주요 사용자**:
  - **일반 사용자(출판사 담당자 등)**: 예스24 MD와 미팅을 예약하고, 본인의 예약 내역을 조회하는 사용자
  - **시스템 관리자(예스24 MD 및 운영자)**: 예약 접수 건을 확인 및 승인/반려하고 시스템의 기본 데이터(MD 정보, 일정, 공지사항)를 관리하는 사용자

---

## 2. 주요 기능 설명

### [사용자 기능]
- **원웨이(One-way) 예약 플로우**: 사용자의 편의성에 따라 2가지 예약 방식을 지원합니다.
  - `MD 먼저 선택`: 원하는 MD를 먼저 고른 후 → 가능한 날짜 → 시간을 선택하는 방식
  - `날짜 먼저 선택`: 원하는 날짜와 시간을 먼저 고른 후 → 해당 시간에 가능한 MD를 선택하는 방식
- **예약 신청**: 이름, 연락처, 출판사명, 이메일, 미팅 목적 등을 입력하여 미팅을 신청합니다.
- **예약 조회**: 본인의 '휴대폰 번호'를 입력하여 신청한 예약의 승인 상태(대기/승인/반려 등)를 조회할 수 있습니다.
- **다크 모드 지원**: 사용자 기기 환경이나 눈의 피로도에 따라 화면을 어둡게 변경하는 버튼을 제공합니다.

### [관리자 기능]
- **예약 관리 (대시보드)**: 접수된 전체 예약 목록을 '날짜별', 'MD별'로 그룹화하여 확인하고, 예약을 승인/거절/취소 처리할 수 있습니다.
- **스케줄 관리**: 선택한 날짜(월~목)와 선택한 MD의 시간대별 예약 가능 여부(오픈/마감/휴가)를 일괄 제어합니다.
- **MD 관리**: 미팅 대상이 되는 MD의 기본 정보(이름, 팀, 카테고리, 상태 등)를 추가, 수정, 삭제합니다.
- **기타 관리**: 시스템 메인 화면에 노출될 '공지사항'을 작성하고, 특정 악성 사용자를 차단하기 위한 '블랙리스트'를 관리합니다.

---

## 3. 사용자 흐름 (User Flow)

### 일반 예약 흐름
1. **메인 페이지 접속**: 시스템 접속 직후 공지사항 및 운영 규칙(오시는 길 등) 확인
2. **예약 흐름 선택**: `MD 우선` 또는 `날짜 우선` 예약 방식 클릭
3. **단계별 선택**: MD 정보 및 희망 일정(날짜/시간) 선택
4. **정보 입력**: 예약자 정보 폼 작성 (팝업 모달)
5. **예약 완료 대기**: 예약 신청 완료 (이 단계에서 예약 상태는 '예약신청(대기)' 상태가 됨)

### 예약 조회 흐름
1. 우측 상단의 **예약 내역 조회** 클릭
2. 본인의 휴대폰 번호 입력 후 **조회** 클릭
3. 접수된 미팅 내역 및 현재 상태(확정, 반려 등) 결과 리스트 확인

## 4. 화면 구성 설명

### 1) 사용자 화면 (`index.html`)
- **주요 UI**: 
  - 상단 브랜딩 (로고, 다크모드 토글, 예약 조회 팝업 버튼)
  - 공지사항 영역 & 안내사항 영역
  - 예약 플로우 진행 영역 (단계별 상태바, MD 카드 목록, 달력 컴포넌트, 시간대 슬롯)
- **역할**: 마법사(Wizard) 형태의 스텝 방식을 제공하여 사용자가 쉽게 단계를 따라 예약을 완료할 수 있게 돕습니다.

### 2) 관리자 대시보드 (`adminV2/index.html`)
- **주요 UI**:
  - 좌측 사이드바 (예약 관리, 스케줄 관리, MD 관리, 공지사항, 블랙리스트 메뉴탭)
  - 메인 데이터 테이블 (상태 필터링, 기간 조회, 검색 지원)
  - 상세 정보 클릭 시 정보를 보여주는 오버레이 모달 창
- **역할**: 모든 데이터를 종합적으로 관리. 특히 예약 확정 및 거절 처리를 원클릭으로 할 수 있는 인터페이스를 제공합니다.

## 5. 시스템 구조

- **프론트엔드 (Frontend)**:
  - React/Vue 등의 최신 프레임워크를 사용하지 않고 **순수 HTML, CSS, jQuery(ES5)** 기반으로 구축되었습니다. (레거시 시스템 호환 및 가벼운 동작 환경 구성)
  - 카카오/네이버 지도 API 라이브러리가 연동되어 '오시는 길'을 시각적으로 제공합니다.
- **API 및 데이터 처리**:
  - 현재 시스템은 백엔드 서버 없이 프론트엔드에서 **JSON Mock 데이터** (`data/*.json`) 파일을 비동기(Fetch/AJAX)로 읽어와 화면에 반영하는 구조입니다.
  - 데이터 변경 및 저장은 현재 메모리 상(Javascript 전역 변수)에서만 수정되며, 실제 서비스 오픈 시 백엔드 REST API와의 연동이 필요합니다.

## 6. 데이터 모델 설명 (JSON 구조 기준)

시스템 동작의 핵심이 되는 주요 데이터 테이블(파일)과 필드 구성입니다.

### 1) `mds.json` (MD 정보 모델)
미팅의 주체가 되는 예스24 내부 담당자 정보입니다.
- `Md_No`: MD 고유 식별 번호 (예: md-01)
- `Md_Name`: MD 이름
- `Team_Name` / `Category_Name`: 소속 부서 및 담당 카테고리 (예: 소설/시/희곡)
- `Status`: 현재 상태 (활성화 여부)

### 2) `bookings.json` (예약 정보 모델)
사용자가 신청한 미팅 예약의 상세 데이터입니다.
- `Booking_Seq`: 예약 건의 고유 식별 번호
- `Md_No`: 해당 예약을 담당하는 MD 번호
- `Meeting_Datetime`: 예약 신청된 미팅 일시 (YYYY-MM-DDTHH:mm:SS)
- `User_Name` / `User_Phone` / `User_Affiliation` / `User_Email`: 신청자 신상 정보
- `Status`: 예약 진행 상태 (PENDING(초기/대기), APPROVED(승인), REJECTED(거절), CANCELLED(취소))

### 3) `slots.json` (시간 슬롯 정보 모델)
특정 MD의 특정 시간대 예약 가능 여부를 원천적으로 통제하는 데이터입니다.
- `Slot_Seq`: 슬롯 고유 번호
- `Start_Datetime` / `End_Datetime`: 슬롯의 시작 및 종료 시간
- `Status`: 상태 (OPEN(예약가능), BOOKED(상담예약됨), CLOSED(관리자에 의한 예약막힘))

*(이 외 시스템 관리를 위해 `notices.json`(공지사항), `blacklist.json`(예약 차단 번호) 모델이 포함되어 있습니다.)*
