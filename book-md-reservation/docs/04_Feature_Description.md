# 프론트 페이지와 어드민 페이지의 주요 기능 설명

YES24 MD 미팅 예약 시스템은 크게 외부 방문자가 예약을 신청/조회하는 `Front-End(사용자 UI)`와 담당 부서가 예약 내역을 관리하고 시스템을 세팅하는 `Admin(관리자 UI)`으로 이분화되어 있습니다.

---

## 1. 프론트 페이지 (사용자 페이지 - `index.html`, `my-bookings.html`) 

사용자 측면에서는 회원가입 절차 없이 빠르고 쾌적하게 미팅을 픽스하는 데 초점을 맞췄습니다.

### 1.1 투-트랙(Two-Track) 예약 방식 설계
사용자의 목적에 맞춘 자유로운 진입점 제공
*   **MD 우선 선택**: 만나고 싶은 특정 분야의 도서 MD가 명확한 사용자들을 위한 진입점입니다. MD를 누르면 해당 MD가 가능한 날짜 캘린더가 켜집니다.
*   **날짜 우선 선택**: 본인의 방문 일정이 고정되어 있어 일정에 맞춰 뵐 수 있는 MD를 찾으려는 출판사/관계자를 위한 진입점입니다.

### 1.2 Step-by-Step 화면 전환 인터페이스
복잡한 수직형 스크롤 폼 대신 한 화면에서 카드 넘기듯 전환되는 모달 방식의 입력 폼을 제공합니다. 
*   진행도 막대(Progress Dot)를 통해 자신이 어느 단계인지 인지할 수 있습니다.
*   시간표 및 캘린더 제어 UI에서 `holidays.js`와 연계해 공휴일과 주말은 아예 선택이 불가능하도록 차단된 시각적 피드백(회색 표시 불가)을 제공합니다.

### 1.3 내 예약 조회 기능 (`my-bookings.html`)
인증 과정 없이 예약자의 휴대폰 번호(혹은 접수 번호)만으로 조회하는 간편 조회 페이지입니다. 접수된 안건의 상태(승인 대기, 예약 확정, 취소 등)를 표기해주어 이메일을 놓친 고객도 언제든 방문 일정을 확인할 수 있습니다.

### 1.4 다크모드/반응형 (테마/디바이스 대응)
*   **다크모드 토글:** 상단의 달/해 아이콘을 눌러 실시간으로 테마 스위칭이 가능합니다. 눈부심 방지가 필요할 때 유용합니다.
*   **모바일 최적화:** 네이버 지도가 축소/확대 문제없이 자연히 모바일에 맞춰지며, 캘린더 타일이나 MD 리스트가 한줄에 한 개씩 떨어지도록 CSS로 미디어쿼리가 대응되어 있습니다.

## 2. 어드민 페이지 (관리자 대시보드 - `adminV2/dashboard.html` 등)

관리자의 운영 생산성을 높이기 위해 넓은 화면을 밀도있게 사용하는 구조입니다.

### 2.1 통합 예약 현황 대시보드
어드민 메인에 접속하자마자 가장 필수적인 핵심 통계수치를 상단에 요약하여 보여줍니다.
*   **일정 요약 KPI:** 오늘 예정된 미팅 건수, 당월 승인 대기 중인 신규 신청 건, MD별로 마감된 비율 등을 한눈에 모니터링합니다.
*   **최근 예약 타임라인:** 하단 Grid에 최근 접수된 실시간 예약 리스트를 띄우고 상태별 태그(대기, 확정, 취소등)를 달아줍니다.

### 2.2 예약 승인 및 상태 제어
사용자가 접수한 '안건'과 '내용'을 시스템 내에서 확인한 뒤 미팅의 확정 여부를 제어합니다.
*   상세 팝업을 열어 출판사명, 전화번호, 미팅 안건을 검토하고 [예약 승인], [거절 사유 입력 후 취소] 버튼을 통해 DB의 상태값을 업데이트합니다.

### 2.3 스케줄 및 휴일 캘린더 세팅 (`admin-schedule.html` 영역)
연차, 워크샵, 공휴일 등 시스템 캘린더를 제어하는 컨트롤 타워입니다.
*   특정 날짜를 선택하여 "임시 휴무일 지정" 버튼을 통해 DB에 휴무 테이블을 인서트 합니다. 이는 앞서 본 프론트엔드의 `holidays.js` 배열 소스와 연동되어 즉시 사용자 화면에서 예약을 막습니다.

### 2.4 공지사항 관리
미팅 시간 변경 등의 중요한 안내를 사용자 페이지 `index.html` 공지사항 띠 배너로 내보낼 수 있는 CMS(Content Management System) 게시판 형태의 페이지입니다.
