# 향후 고도화 가능한 기능 요약 (Future Roadmap)

본 프로젝트는 현재 MVP(Minimum Viable Product) 수준의 초기 모델과 프론트엔드 퍼블리싱 및 클라이언트 로직 구현이 1차적으로 완료된 상태입니다. 실 운영 이후 시스템 활용성을 극대화하기 위해 추천하는 향후 고도화 플랜입니다.

---

## 1. 예약자 노쇼(No-Show) 대비: 자동 푸시 및 리마인드 알림 연동
단순 예약 확정 메시지 발송에 그치지 않고, 시스템 외부 연동을 통해 미팅 도달률을 높입니다.
*   **카카오 알림톡 / SMS API 연동:** 예약 승인 템플릿, 거절 사유 템플릿을 나누어 관리.
*   **전날 자동 리마인드 스케줄러 기능:** D-1 낮 12시에 '내일 오후 14:00 미팅이 예정되어 있습니다.' 등의 알림을 배치(Batch) 프로그램으로 쏘는 기능
*   **노쇼 블랙리스트 관리:** 패널티를 받은 핸드폰 번호를 MD가 블랙리스트 UI에 등록하여, 해당 연락처로의 예약을 원천 차단하는 방어 기능.

## 2. 동시성 (Concurrency) 보호 : 다중 예약 충돌 방지 로직
시스템 오픈 직후 인기 많은 MD의 특정 시간대에 다수의 사용자가 동시에 예약을 시도할 경우 발생하는 기술적 리스크입니다.
*   **낙관적 락(Optimistic Lock):** 예약 완료 버튼이 눌리는 순간, 해당 타임 슬롯 버전이 이미 선점되었는지 DB에서 확인 후, 빼앗겼다면 즉시 "방금 예약이 찼습니다"라고 사용자 프론트에 반환하는 방어 로직 구현.
*   **슬롯 홀딩(Holding) 타임:** 사용자가 입력 폼 모달을 연 순간, 결제 시스템처럼 약 5분간 임시 락(Temp Lock)을 걸어 다른 사용자가 해당 슬롯을 클릭하지 못하게 막아주는 시스템 고도화.

## 3. 스마트 스케줄 매칭 (Auto-Matching)
현재는 유저가 MD 정보를 일일이 확인하거나 무작위로 빈 시간을 찍어봐야 하지만, 조건 기반 매칭 로직을 제공할 수 있습니다.
*   "내가 취급하는 카테고리(예: IT/개발 서적)와 내가 서울에 오는 날짜(예: 11월 15일 오후)"를 콤보 박스로 선택해 [검색] 버튼을 누르면, **조건에 모두 부합하는 비어있는 MD 리스트를 한 번에 필터링해주는 뷰(View) 추가**.

## 4. 어드민: 엑셀 다운로드 및 통계 심화
운영 효율화를 위한 관리자 전용 통계 기능을 강화합니다.
*   월별 어느 분야의 MD 미팅이 제일 활발했는지 시각화하는 차트 개발(Chart.js 추가).
*   특정 날짜 범위 내 예약자 정보를 엑셀 `.xlsx` 등으로 일괄 다운로드 할 수 있는 클라이언트 사이드 변환 모듈 삽입 도입.

## 5. 관리자 접근 제어(SSO) 및 JWT/세션 보안 (필수)
현재의 정적 HTML 어드민 구성물들을 실제 서버로 이전 시 0순위로 진행해야 하는 고도화입니다.
*   YES24 내부 Active Directory 아이디나 기존 사내 그룹웨어 계정 인증 API (Single Sign-On)과 연계.
*   비인가 접근 시 로그인 페이지 리다이렉션 방어 코드 작성 및 쿠키/토큰 베이스 통신.
