# 주요 기능별 프롬프트 요약 (Prompt Engineering Log)

본 문서는 프로젝트를 진행하면서 주요 기능 구현 시 AI에게 요청했던 기획 의도와 해결 전략을 담은 프롬프트 패턴 기록입니다.

## 1. 초기 시스템 설계 및 UI/UX 구조화
*   **요청 목적:** 디자인 가이드를 준수하면서 백엔드 ASP.NET 환경에 적합한 레거시 프론트엔드 구성
*   **프롬프트 패턴:**
    > "YES24 MD 미팅 예약 시스템의 사용자 페이지를 작성해줘. 프레임워크(React, Vue 등)는 배제하고 순수 HTML, CSS, jQuery 기반(ES5)으로 단일 파일 내에서 작동하도록 작성해야 해. 디자인은 Stitch 가이드를 따르며, 반응형으로 만들어줘."
*   **해결 전략:** 불필요한 빌드 파이프라인 없이 서버에 바로 올릴 수 있도록 바닐라 JS 객체 분리와 모듈화를 시도했습니다.

## 2. 모달 기반의 단계별 예약 플로우(Step-by-step Flow)
*   **요청 목적:** 한 페이지에서 사용자가 끊김 없이 MD ➡️ 날짜 ➡️ 시간을 선택할 수 있는 UX 구현
*   **프롬프트 패턴:**
    > "기존 예약 폼을 화면 전환(Step-by-step) 방식의 플로우로 리팩토링해줘. 각 단계별 전환 시 투명도(Opacity)가 변하는 CSS 애니메이션을 적용하고, '이전', '취소' 버튼을 통해 상태 초기화 및 네비게이션이 가능하게 해줘."
*   **해결 전략:** 플로우 컨테이너 내에서 `active` 클래스를 토글하여 화면을 전환하고, 진행도 닷(Dot) UI를 추가하여 현재 위치를 시각화했습니다.

## 3. 스케줄링 및 공휴일 예외 처리 로직
*   **요청 목적:** 주말 차단 및 명시된 공휴일(배열)에 예약이 불가능하도록 캘린더 연동
*   **프롬프트 패턴:**
    > "예약 가능 요일과 시간을 설정 객체(`SCHEDULE_CONFIG`)로 분리해줘. 그리고 `holidays.js` 모듈을 따로 빼서 한국의 법정 공휴일 날짜와 주말에는 달력의 날짜 버튼이 비활성화(Disabled) 되도록 달력 생성 로직을 맵핑해줘."
*   **해결 전략:** 달력 렌더링 시 매 일자마다 `isHoliday()` 함수를 통과시키고, 요일 인덱스를 통해 주말을 차단하는 방식으로 무결성을 확보했습니다.

## 4. 다크모드(Dark Mode) 지원
*   **요청 목적:** 현대적인 웹 스탠다드인 다크모드 토글 버튼 추가
*   **프롬프트 패턴:**
    > "웹페이지에 다크모드 토글 기능을 추가해줘. `dark-mode.css` 파일을 별도로 생성해서 색상 오버라이딩을 진행하고, 사용자가 토글 버튼을 클릭할 때 해당 CSS가 HTML 문서에 동적으로 삽입되거나 해제되게 해줘."
*   **해결 전략:** 색상 테마 분리를 완전히 독립 파일로 빼냄으로서 기존 라이트모드 CSS(`app.css`)의 코드 오염을 막았습니다.

## 5. 어드민 대시보드 구조 최적화 (Admin UI)
*   **요청 목적:** 관리자가 방대한 예약/MD 데이터를 한눈에 볼 수 있도록 반응형 전체화면 구성
*   **프롬프트 패턴:**
    > "어드민 페이지의 우측 대시보드 영역이 좁은 디스플레이나 아주 넓은 모니터에서도 화면 전체를 꽉 채우도록(Full-width) CSS Flex 구조를 활용해 리팩토링해줘. LNB(사이드바)는 고정폭을 유지해야 해."
*   **해결 전략:** `display: flex` 부모 하위에서 사이드바 `width`를 픽스하고 메인 컨텐츠에 `flex: 1`을 부여하여 남은 브라우저 너비를 모두 차지하도록 적용.
